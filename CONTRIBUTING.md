# 点灯蛙 (DianWa) 社区共建指南与开发协议 (CONTRIBUTING)

首先，感谢您关注并愿意参与家庭教育防干扰的开源运动。点灯蛙 (DianWa) 不仅仅是一个代码库，它是所有焦虑家长联合起来，**抵御低质量AI内容对孩子造成直接干扰和思维降级**的一场反击战。

为了保证生态中流通的每一个组件（Prompt、考试模板、MCP Skill等）都能真正帮助到孩子，我们在合并您的 Pull Request (PR) 前，会做严格审查。

无论您是写代码的极客老爸，还是不碰代码的一线教师，我们都极其欢迎！

## 💡 我不是程序员，我该如何贡献？
我们最缺的不是冷冰冰的代码，而是**"懂孩子的灵魂追问"**。

1. 在 `K12-learning-prompts/` 目录下，找到您擅长的学段文件夹：
   - 📁 `小学低年级_1-3年级_兴趣保护版.md`（1-3年级）
   - 📁 `小学高年级_4-6年级_会学能力版.md`（4-6年级）
   - 📁 `初中_7-9年级_中考意识版.md`（7-9年级）
   - 📁 `高中_10-12年级_元认知版.md`（10-12年级）

2. 提交一个 Issue（我们设置好了专属模板），写下：当孩子遇到某个类型的难题时，您觉得**最有效的一句安抚话 + 最巧妙的一个追问是什么**。

3. 我们的社区开发者会把您的这句"灵魂之问"封装进系统模板。您的头像将永久展示在名人堂！

## 💻 极客开发者与插件开发者投稿指南 (Diandengwa Protocol)

如果您开发了一个连接智能音箱的 Adapter，或者编写了一套"满分作文分析 MCP 引擎"，请确保它**绝对服从以下的底线标准**：

### 1. 核心协议 (The Socratic Rule) 🛑
**所有大模型的最后输出，严守"启发式追问"原则。**
所有的贡献代码或 Prompt 模板链：
- **严禁**直接输出最后一步解题步骤。
- **严禁**提供纯粹的正确代码或全文标准译文。
- **必须**在每一轮对话结束时，把思考的接力棒以一个微小的测试题/提问交还给孩子。

### 2. Schema Standards (打标规范)
您的工具文件头部，必须包含：
- `@DiandengwaLevel`: 如 `#初中/几何`（禁止写笼统的词汇）
- `@DiandengwaTag`: 当前工具针对哪类错因（如 C类-方法缺失、H类-情绪干扰）
- `@DiandengwaVersion`: 当前版本号

### 3. PR 评审（盲测试跑机制）
1. 在提交您的 PR 前，您**必须**在真实的模型平台上（豆包/ChatGPT/内部跑的脚本）运行这一套提示词或工具。
2. 在您的 PR 具体描述里，请附上**至少三轮**由于孩子犯错而触发的、您的工具巧妙引导解决的**真实对话 Log**。
3. 点灯蛙陪审团会进行人工验证。成功 Merged 的内容，将获得 `Diandengwa Certified` 官方认证！
